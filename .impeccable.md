## Design Context

### Users
Developers joining or working on unfamiliar codebases. They're in VS Code, mid-task, and need to understand how code connects — not read documentation. Context is professional but informal: side project contributors, new hires onboarding, engineers debugging across team boundaries.

### Brand Personality
**Friendly, Curious, Playful** — like a knowledgeable friend who makes exploring code genuinely enjoyable. Not a formal teacher, not a corporate tool. Has opinions, celebrates your progress, makes the mundane feel engaging.

### Aesthetic Direction
**Visual tone:** Clean and compact like Linear/Raycast, but with warmth and personality. Keyboard-driven, fast, feels native to VS Code — not a bolted-on web app. The confetti and animations are earned moments, not decoration.

**References:**
- **Linear** — speed, keyboard-first, understated animations, dark-by-default confidence
- **Raycast** — compact utility, beautiful details in small spaces, feels like a native OS tool
- **GitHub Copilot Chat** — lives naturally in VS Code, explanatory, inline
- **Arc Browser** — opinionated, breaks conventions thoughtfully, bold visual identity

**Anti-references:**
- **Jira / enterprise tooling** — heavy, slow, over-configured, corporate. Feels like work about work.
- **Overly gamified apps** — too many badges, points, streaks. Feels childish for professional developers.
- **Generic AI chat** — bland, conversational, no structure. Just a text box with no opinion.

### Design Principles

1. **Native, not bolted on** — Every surface should feel like it belongs in VS Code. Use theme tokens, standard patterns, codicons. If a user can't tell where VS Code ends and Side Bae begins, that's a win.

2. **Explain through structure, not just text** — Graphs, colored borders, spatial layout, and visual hierarchy should do as much explaining as the words. A glance at the card should convey meaning before reading.

3. **Celebrate without gamifying** — Confetti on completion, sparkle on leaf nodes, warm copy ("nice work") — these are earned moments of delight. But no points, no streaks, no badges. The reward is understanding the code.

4. **Fast and forgettable** — The interface should be fast to use and easy to forget. Navigation shouldn't require thought. The user's attention belongs on the code, not the tool. If they notice the UI, it's too loud.

5. **Opinionated defaults, minimal configuration** — One button to start, one click to navigate, one flow per mode. Don't ask users to configure what you can decide for them. The extension should just work.

### Technical Design Tokens
- **Colors:** Exclusively VS Code theme tokens (`--vscode-*`). No hardcoded hex values.
- **Typography:** Editor font family for code, system font for UI. Scale: 10/11/12/13/14px.
- **Spacing:** 4px base unit. Gaps: 4/8/10/12px. Padding: 8/12/16/20px.
- **Border radius:** 3-4px consistently.
- **Animations:** 0.15-0.2s ease-out for entrances. Respect `prefers-reduced-motion`. Confetti is opt-out via settings.
- **Breakpoints:** 320px (narrow), 500px (wide). Content-driven, not device-driven.
